# TaskBarEx

Windows 11のタスクバーを拡張する Qt6 アプリケーション

## 概要

Windows 11では1段タスクバーに制限されて使いにくくなりました。TaskBarExは、タスクバーの表示と同期してすべての実行中ウィンドウを一覧表示するアプリケーションです。

### 主な機能

- **リアルタイム ウィンドウ表示**: 実行中のすべてのアプリケーション/ウィンドウを表示
- **Windows風デザイン**: タスクバーと同様の見た目と操作感
- **自動表示・非表示**: マウス座標ベースのシンプルな表示制御
- **クリック操作**: ボタンクリックでウィンドウをアクティブ化
- **グループ化**: 同一アプリケーションのウィンドウを自動グループ化
- **DPI対応**: 高解像度ディスプレイでの適切なサイズ調整

## 動作仕様

### 表示・非表示ロジック

TaskBarExは以下のマウス座標に基づいて自動的に表示・非表示が切り替わります：

| マウスY座標 | 動作 | TaskBarEx |
|------------|------|-----------|
| **Y >= 1070** | 画面下端から10px以内 | **即座表示** |
| **Y 984-1070** | 表示維持範囲（86px幅） | **表示継続** |
| **Y < 984** | アプリバー上端より上 | **即座非表示** |

### 操作方法

1. **表示**: マウスを画面下端（Y >= 1070）に移動
2. **ウィンドウ切り替え**: TaskBarEx上のボタンをクリック
3. **表示維持**: マウスをアプリバー領域（Y 984-1070）内で操作
4. **非表示**: マウスをアプリバー上端（Y < 984）より上に移動

## インストールと実行

### 必要要件

- **OS**: Windows 10/11
- **ランタイム**: Qt6ランタイム（自動インストール）
- **画面解像度**: 1080p以上推奨（座標ベース表示制御）

### 実行方法

1. **ダウンロード**: 最新のTaskBarEx.exeを取得
2. **実行**: ダブルクリックで起動
3. **確認**: マウスを画面下端に移動して動作確認

```bash
# 開発環境での実行
cd TaskBarEx
./build/TaskBarEx.exe
```

## トラブルシューティング

### よくある問題

**Q: TaskBarExが表示されない**
A: マウス座標が表示閾値に達していない可能性があります。Y座標1070以上（画面下端から10px以内）に移動してください。

**Q: マウス移動に反応しない**
A: 座標ベース検出の問題です。1080p画面でY座標984-1070が有効範囲です。

**Q: アプリケーションが起動しない**
A: Qt6ランタイムが不足している可能性があります。自動DLLコピー機能で解決済みです。

### ログ確認

詳細なログは以下で確認できます：

```bash
# 最新ログファイル
ls -lt build/log/TaskBarEx_*.log | head -1

# リアルタイムログ監視
tail -f build/log/TaskBarEx_*.log
```

## 開発情報

### ビルド方法

```bash
# 開発環境セットアップ
export PATH="/c/Qt/6.10.0/mingw_64/bin:/c/Qt/Tools/CMake_64/bin:$PATH"

# ビルド実行
cmake -B build
cmake --build build

# 実行
./build/TaskBarEx.exe
```

### 技術仕様

- **フレームワーク**: Qt6.10.0 (C++)
- **Windows API**: EnumWindows, SHGetFileInfo, GetWindowRect
- **特殊機能**: マウス座標ベース表示制御、循環参照防止、高精度トラッキング（50ms間隔）
